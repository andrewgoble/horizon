## Executive Summary

Jambys ships **485 theme-check errors** (307 are missing translations, 114 are missing img width/height on Replo dead code — but **29 `CaptureOnContentForBlock` errors** and **11 broken Rivo blocks** are real bugs touching live pages).

Three patterns dominate the issue surface:

1. **Dead-code drag:** 17 unused Replo section shells + 5 inactive Shoplift `sl-*` sections + 9 Shoplift snippets + 2 unused Wonderment sections + a 6-LOC orphan cart-upsells alias and a 151-LOC orphan filters-toggle = ~**9,400 LOC and ~5 MB** of theme code that ships but never executes. Deleting this clears 90%+ of theme-check noise and removes a `unicons.iconscout.com` render-blocking external CSS request that the orphan sections still reference.
2. **Performance is single-bug-dominated:** the **homepage LCP image at `blocks/image.liquid:36-50` has no `fetchpriority="high"`, no `loading="eager"`, and no `<link rel="preload">`**. That, plus the synchronous Typekit `<link rel="stylesheet">` at `snippets/fonts.liquid:16`, explains most of the 10.2s mobile LCP. Both are 1-day code fixes worth ~2–4s LCP.
3. **Accessibility regressions persist from baseline:** the slideshow `aria-hidden` + focusable-children pattern remains (`assets/slideshow.js:165, 181, 253, 740`); heading hierarchy still skips (h1→h6 on home, h1→h4 on PDP); `.compare-at-price` opacity 0.4 fails contrast site-wide; a real product link is `aria-hidden="true"` in `blocks/product-title.liquid:11`.

**Recommended posture:** REDUCTION first (delete dead code, fix theme-check explosions), then T1 critical bugs, then perf wins.

## C3. LCP Path Analysis

Homepage `templates/index.json` LCP element is `shopify://shop_images/home-page-new-photo.jpg` rendered through `blocks/image.liquid`.

**Critical bug:** `blocks/image.liquid:36-50` calls `image_tag` **without `loading: 'eager'` and without `fetchpriority: 'high'`**. Preload scanner defaults the LCP image to lazy / auto-priority — deprioritized below DOM order.

`sections/hero.liquid:54-58, 74` correctly wires `fetch_priority = 'high'` when `section.index == 1`. The live homepage uses a generic `image` block, not the `hero` section. The block must accept `fetchpriority` + `loading` settings or detect first-position automatically.

No `<link rel="preload" as="image">` for the hero exists anywhere. Adding one is the single biggest LCP win.

**Font-display:** zero `font-display: swap` declarations anywhere in `assets/*.css`. Code-side override: add `@font-face { font-family: 'tk-…'; font-display: swap; }` redeclarations in `base.css` after the Typekit `<link>`.
